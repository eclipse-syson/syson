= 2026.5.0 (work in progress)

== Key highlights

* SysON now requires *Java 21* or later.
* New diagram tools let you create timeslices and snapshots from `OccurrenceUsage`, `ItemUsage`, and `PartUsage` graphical nodes.
* Improved multi-selection support in standard diagrams for _Duplicate Element_, _View As_, and _Add existing elements_. _Duplicate Element_ now also has the `Ctrl+d` key binding.
* Improved selection dialogs in several creation tools: some specialization or reference selections are now optional, and standard library candidates are grouped under a `Libraries` category.
* In the _Explorer_ view, `Expression` elements now display their full textual representation.

== Breaking changes

* As of version 2026.5.0, SysON now requires *Java 21* or later.

== New features

* In diagrams:

** Add tools to create _timeslices_ and _snapshots_ on `OccurrenceUsage`, `ItemUsage`, and `PartUsage` graphical nodes.
These tools use a selection dialog to either create a graphical node based on an existing _timeslice_ or _snapshot_, or, if nothing is selected, create a default _timeslice_ or _snapshot_ `OccurrenceUsage` graphical node.

** Improved support for `Stakeholder`.
It is now possible to create new _stakeholders_ from `Requirement` and `Concern` graphical nodes, whether they are definitions or usages.
Created stakeholders must reference an existing `PartUsage`.
By default, stakeholders are represented as dedicated graphical nodes connected to their parent graphical node, but they can also be displayed in the `stakeholders` compartment.

** Add support for the _frame_ graphical edge.
A frame edge can be created between a `RequirementUsage` or a `RequirementDefinition` graphical node and a `ConcernUsage` graphical node.

== Bug fixes

* In diagrams:

** In _Interconnection View_ diagrams, fix an issue where the `parts` compartment of a `PartDefinition` graphical node was incorrectly revealed when creating a `PartUsage` from that `PartDefinition`, even when the `interconnection` compartment was already visible.

* In the _Details_ view:

** Fix an issue that allowed users to edit library objects from the _Details_ view.

* In the _Explorer_ view:

** When creating a `ConstraintUsage` inside a `Requirement`, the new constraint is now correctly owned through a `RequirementConstraintMembership`.

== Improvements

* In diagrams:

** Improve the tool for creating an `Actor` graphical node by making specialization selection optional.
** Improve the tool for creating a `Stakeholder` graphical node by making specialization selection optional.
** Improve the tool for creating a `Subject` graphical node by making specialization selection optional.
** Improve the tool for creating a `FlowUsage` from a `ConnectionUsage` by making payload selection optional.
** Merge the two _objective_ creation tools into a single tool by leveraging the updated selection dialog, which makes specialization selection optional.
** Merge the two _perform action_ creation tools into a single tool by leveraging the updated selection dialog, which makes selection of a referenced action optional.
** Improve the _Duplicate Element_ tool to support multi-selection in standard diagrams, and add the `Ctrl+D` key binding.
** Improve the _View As_ tool on graphical nodes to support multi-selection in standard diagrams.
** Improve the _Add existing elements_ tool on graphical nodes to support multi-selection in standard diagrams.
** Improve the tools for creating `PerformActionUsage` by making selection of a subsetted reference optional.
** Improve the tool for creating a `SatisfyRequirement` graphical node, whether standalone, feature typed, or subsetting a `RequirementUsage` by reference.
** Reorganize the three tools for creating `ExhibitStateUsage` graphical nodes into two tools: one for parallel `ExhibitStateUsage`, and one for non-parallel `ExhibitStateUsage`.
Both tools optionally allow the selection of an existing `StateUsage` to exhibit.
** Improve the selection dialog used by various tools by grouping all standard library candidates under a `Libraries` category.

* In the _Explorer_ view:

** `Expression` elements now display their full textual representation:
